One re-preprocess + one fresh train. Three intended changes; only B and C shipped (A reverted).

## A. Slider velocity (SV) — REVERTED (the decode-side approach was wrong)

The first attempt derived SV per-slider from geometry/duration and "sectioned" it — ~24 SVs / 77
points (user: *"terrible"*). **Why it's wrong:** SV is **not** a per-slider geometric consequence;
it's a **structural/stylistic** choice mappers make in a few **coarse sections, like kiai** (slow part
→ low SV; drop → SV≈1+; most of the map ≈1; occasional fast 1–6 s burst). Real maps have **few** SV
sections, tied to song structure. Reverted to clean SV=1.

**The right way (became v7):** SV must be **learned/structural** — an SV channel the model learns (it
reproduces where real mappers put coarse SV changes, naturally sparse). See [v7.md](v7.md). The
durable lesson is in [knowledge/lessons-learned.md](../knowledge/lessons-learned.md).

## B. adaLN-zero conditioning — DONE

Replace the additive FiLM (`h += time(t_emb)`) with **DiT adaLN-zero**: each `ResBlock1d` predicts
`(scale, shift, gate)` from the conditioning embedding (`SiLU→Linear`, gate **zero-init** so blocks
start as identity), modulating `h = norm(h)·(1+scale)+shift` and gating the residual `x +
gate·block(...)`. Gives difficulty multiplicative control; the 2026-survey "contained upgrade".
Model-only change (hermetic-testable); fresh train (can't load v5 ckpt). `--adaln` default on.

(~25,073 maps), 17-ch.

## Outcome

Trained 60 epochs, clean convergence, no divergence. **Static metrics are roughly a wash vs v5.**
- SR calibration tighter to target in the low/mid range (v5 over-shoots everywhere; v6 close except an
  SR4 dip), and more consistent in-range mid-curve.
- v5 has marginally more uniform kiai across SRs and slightly higher hitsounds.
- **One flag: kiai 0.00 at SR4** (others 0.10–0.21) despite 100%-kiai gold data — the
  borderline-section-flips-per-sample problem that motivates the planned supervised kiai head.

The adaLN difficulty-control + gold-data kiai/hitsound consistency wins are the kind that show up in
**play**, not corpus stats. The next batch (v7) targeted the residue: same-speed sliders (the SV
channel), inconsistent kiai, weak hitsounds, density gaps.

## Architecture context (2026 survey)

The field moved U-Net → **DiT** (pure transformer, adaLN-zero, RoPE, QK-norm). We already have MHSA +
QK-norm; the U-Net **long skip connections** are the critical part (ablations show even U-ViT/DiT keep
them). Full DiT shines at scale we don't have; the 2026 consensus is "data + training > arch tricks"
(matches the ranked-data win). The **contained upgrade worth trying** was adaLN-zero (this batch) —
RoPE in attention was a smaller second (tried at v7, see [v7.md](v7.md)).
