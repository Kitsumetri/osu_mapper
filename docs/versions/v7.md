**Root cause (key):** patterns (#3) and straight sliders (#5) are the **same bug — under-dispersion
from ε-MSE** (the model regresses spatial outputs toward the mean → compressed spacing *and* collinear
slider anchors). **Flow angles are already ≈ real → attention is NOT the bottleneck**;
objective/representation > attention. Real SV is **structured + sparse** (~5 vals / ~10 changes per
map) → validates a learned SV channel. Curve target set to **38–45%** visibly-curved.

### P4 channel detail
- **SV channel (highest priority).** Per-frame continuous channel = the effective SV-multiplier
  timeline, `log2(SV)` scaled+clamped to ~[0.25×,4×]→[−1,1]. Decode — **stability over fidelity**:
  median-filter → quantise → hysteresis (ΔSV ≥ ~0.15) → min section ~1–2 s → hard-cap ~8 sections.
  Slider duration = `pixel_len/(100·SliderMultiplier·SV)·beat`.
- **Curve cue.** Per-slider intended-sagitta scalar held over the span, so the model signals
  curve-vs-straight intent decoupled from anchor MSE.

## v7-vpred (17-ch) — Phase 2 result

v-pred closed ~70% of the mean-spacing gap and ~28% of the jump gap (bigger, more confident
movements), but **did NOT improve spacing variety, streams, or slider curvature** (median slider still
dead-straight). Conclusion: the objective fixes average *magnitude*, not *variety*/curvature → P4
flow/curvature channels justified.

## v7-full (19-ch) — SV + curve + RoPE + up-attn

✅ The **SV channel works** (0→5 sections/map, 100% non-trivial). ✅ **curvature jumped** 13→29%. ✗
But **spacing/jumps REGRESSED hard** (jump 0.145→0.048, mean-spacing 129.6→102.8, turn 88→77). Spacing
is object *positions* → model-side. **Prime suspect: the attention add (rope/up-attn).** Bundling
P2+P3+P4 lost attribution.

## v7.5 (20-ch) — drop attention + red corners. BEST so far

### Why `--up-attn` killed jumps (the lesson)
**Self-attention is a weighted *average* across time** — it pulls each frame's cursor value toward an
attended mean, i.e. it *smooths* the spatial channels and reduces their variance. Jumps need the
opposite. **Up-path** attention sits right before the output, so its smoothing hits the final positions
directly. The turn-angle drop (88→77 = clustered) is the fingerprint. Matches Phase 1: flow angles
were already ≈ real, so attention was not the bottleneck — adding it *actively hurt*. → **Leave
`--up-attn` off** ([knowledge/lessons-learned.md](../knowledge/lessons-learned.md)).

### Red vs white slider points → the `corner` channel (ch 19, 20-ch)
osu! sliders have **white** (smooth) and **red** (sharp-corner) control points; a **red point = a
doubled consecutive control point**. Measured (250 ranked maps): **12.7% of sliders have ≥1 red
point**, ~75% of bezier sliders are angular. The corner cue is a per-slider scalar set from real
red-point detection (detected **before** RDP, which collapses duplicates). Decode: when the cue is
high, emit sharp anchors as red (doubled). `write_osu` writes `curve_points` verbatim, so doubling them
is enough.

### v7.5 design (one retrain)
Drop `--rope --up-attn` (keep SV + curve + v-pred); add the corner cue → 20-ch. **= v-pred's jumps +
v7's SV + curves + red corners.**

### Outcome
Dropping up-attn recovered jump_ratio 0.048→0.131 (≈ v-pred level), mean-spacing and turn-angle back
to ≈ real, **while keeping** the SV channel (~4 stable sections) and curves (28% visible). Red corners
generate but **under-produced (2% vs real ~13%)** — the rare-binary mean-regression problem; decode
knob `CORNER_DECODE_THRESHOLD` is tunable. Best model so far.

**Play feedback (in-game):** rhythm 7/10; hitsounds 4/10 (unstable song-to-song); sliders 6/10 (still
many straight, but red-corner + short/long SV sliders work — new wins); patterns 6/10 (jumps
sometimes good/nasty, streams clearly better). Bug: the last circle sits in the dead outro
("Happppy") → trailing-trim decode fix.

## Loss-function analysis (carried forward to v8)

Diffusion trains on MSE between predicted and true noise (ε or v) — the score-matching objective; we
don't replace it, we swap the pointwise distance (L1/Huber) or add auxiliaries.
- **LPIPS / Gram-literal don't transfer** (no pretrained beatmap perceptual net; our target isn't an
  image; metrics are non-differentiable).
- **Weighted MSE is the most promising** — (a) **per-channel weighting** (the spatial cursor/anchor
  channels are underfit while easy piecewise channels are "solved" early → up-weight spatial); (b)
  active-frame weighting (weight frames near onsets). → became v8's `--spatial-loss-weight`.
- Other low-risk options wired in: **L1/Pseudo-Huber** (`--loss huber`), **Min-SNR-γ** timestep
  weighting (`--min-snr-gamma`). Honest priority: the loss is a *secondary* lever for dispersion;
  representation + avoiding over-smoothing matter more. Try cheap stable nudges one at a time.
